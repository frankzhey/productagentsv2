1. 页面地图

页面地图与关系

核心保持两层结构。首页负责选诗和浏览，详情页负责原文阅读与理解辅助。原型中的桌面双栏只是评审用联动视图，移动端则严格保留真实页面切换方式。

古诗词列表页

展示古诗卡片、搜索入口、快捷筛看和课程范围提示，用户在这里做“选哪一首”的决策。

Banner 区 古诗卡片栅格 搜索与筛看 推荐入口

古诗详情页

聚焦单首古诗，展示原文、注释、译文、赏析、学习提示和返回列表操作。

诗文封面区 学习辅助切换 学习提示 返回列表

2. 用户流程

用户路径与操作步骤

把核心动作压缩到四步，并且每一步都有清晰反馈，方便产品和研发验证交互是否符合儿童认知节奏。

浏览列表

用户进入首页，看到古诗卡片、推荐标签和搜索入口，快速理解这里是“点卡片选诗”的页面。

点击古诗

点击任意卡片后，桌面端即时切换右侧详情，移动端直接进入详情视图。

切换辅助内容

用户按需在注释、译文和赏析之间切换，先读原文，再逐步深入理解。

返回继续学习

移动端可以点击“返回列表”，桌面端可继续点下一张卡片，形成连续学习路径。

3-7. 结构与交互

页面结构、组件、状态与响应式

按 PRD 划分 Header、Content、Action 区，并确保原型中每个动作都能真实触发，避免“看起来有按钮，实际上点不动”的评审误差。

页面结构

Header 放品牌与导航。列表页 Content 放搜索、卡片和筛看。详情页 Content 放原文、辅助内容和学习提示。Action 区统一放按钮、切换入口和返回入口。

核心组件

古诗卡片、诗文头图区、搜索输入框、注释译文赏析切换、状态标签、学习提示区、移动端切屏容器。

交互说明

卡片可点击切换详情，搜索会实时过滤，辅助内容按钮会切换内容面板，移动端可进详情也可返回列表，所有主交互都有激活态和轻量动效反馈。

状态设计

  • 默认态：展示推荐古诗与默认详情内容。
  • Loading：有进入式加载卡，提升打开原型时的沉浸感。
  • 空状态：搜索无结果时给出友好提示，而不是空白。
  • 错误态：正式研发可在数据缺失或路由错误时补兜底提示。
  • 成功态：切换卡片、面板和移动端视图时均有明确反馈。

数据需求与响应式

  • 数据字段：slug、title、author、dynasty、excerpt、lines、annotations、translation、appreciation、tips、recommendations。
  • 依赖方式：当前原型在单文件中用本地 JS 数据模拟，正式实现可平移到 JSON 或 TS 数据源。
  • Desktop：双栏联动，适合评审信息切换效率。
  • Tablet：自然下折为单列，缩短横向扫描距离。
  • Mobile：保留列表与详情的真实切屏逻辑，保证阅读和点击舒适。

      已实现动作

      点击桌面或手机卡片会切换当前古诗。桌面搜索会实时过滤卡片。注释、译文、赏析可以切换。移动端可以进入详情并返回列表。
