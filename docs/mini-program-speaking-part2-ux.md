# Mini Program Speaking Part 2 UX 方案

## 1. 页面地图
- 页面列表
  - Main 口语测评首页
  - Recording Page 开始测评页
  - Waiting for Score 测评状态页
  - Score 测评结果页
  - Activity Ended 活动结束态
- 页面之间关系
  - 首页 -> 开始测评页：用户未提交且活动进行中时可进入
  - 首页 -> 等待页：用户已提交但尚未出分时进入
  - 首页 -> 结果页：用户已出分时进入
  - 首页 -> 活动结束态：活动已结束且用户未提交时展示
  - 开始测评页 -> 等待页：用户上传录音成功后进入
  - 等待页 -> 结果页：系统已收到结果回传后进入

## 2. 用户流程
- 用户路径
  - 进入小程序活动首页
  - 微信授权并完成身份识别
  - 系统判断当前参与状态
  - 未提交用户进入录音页完成录音与上传
  - 已提交未出分用户查看等待文案并可分享
  - 已出分用户进入结果页查看 CEFR 等级与文案
- 操作步骤
  - Step 1：用户打开活动首页并阅读活动说明
  - Step 2：授权成功后系统识别 unionId 并校验是否已参与
  - Step 3：未参与用户点击立即参与或开始口语测评
  - Step 4：系统随机展示 38 套题目中的 1 套题图
  - Step 5：用户点击录音按钮开始 2 分钟倒计时录音
  - Step 6：用户可手动结束录音，也可等待系统自动结束
  - Step 7：录音结束后用户可重新录音、播放或上传录音
  - Step 8：上传成功后用户进入等待页
  - Step 9：结果回传后用户从首页或等待态进入结果页
- 页面跳转
  - 首页 CTA 与状态强绑定，避免用户误入错误流程
  - 活动结束时仅未提交用户停留在结束态，已提交用户仍可查看等待页或结果页

## 3. 页面结构
### Main 口语测评首页
- Header
  - 活动标题
  - 品牌识别区
- Content
  - 主 Banner 区
  - 活动简介区
  - 参与规则卡片
  - 状态说明卡片
- Action
  - 立即参与按钮
  - 开始口语测评按钮
- 状态变体
  - 未提交：两个按钮均可点击
  - 已提交未出分：两个按钮均展示等待测评结果且禁用
  - 已出分：两个按钮均展示查看结果且可点击
  - 活动结束未提交：两个按钮均展示谢谢参与，活动结束且禁用，Banner 切换为结束素材

### Recording Page 开始测评页
- Header
  - 返回按钮
  - 页面标题
  - 录音时长规则提示
- Content
  - 题目图片区
  - 题目说明区
  - 录音状态卡片
  - 倒计时显示区
- Action
  - 开始录音按钮
  - 结束录音按钮
  - 重新录音按钮
  - 播放按钮
  - 上传录音按钮
- 关键状态
  - 默认态
  - 录音中
  - 录音完成待确认
  - 上传中
  - 上传失败

### Waiting for Score 测评状态页
- Header
  - 页面标题
- Content
  - 状态图标或插画
  - 固定文案区
  - 下一步说明区
- Action
  - 分享给好友按钮
  - 分享到朋友圈按钮
  - 返回首页按钮

### Score 测评结果页
- Header
  - 页面标题
- Content
  - CEFR 等级主视觉图区
  - 等级背景图层
  - 等级名称与简述
  - 鼓励文案区
  - IELTS and the CEFR 链接区
- Action
  - 返回首页按钮
  - 再次分享按钮

## 4. 核心组件
- Banner 组件
  - 支持进行中与活动结束两套素材
  - 支持标题、副标题与状态标签
- CTA 按钮组件
  - 主按钮用于立即参与/查看结果
  - 次按钮用于开始口语测评/分享等辅助操作
  - 包含默认、hover、active、disabled 四种状态
- 状态卡片组件
  - 展示当前用户状态、活动状态和流程提示
- 题目图片卡组件
  - 承载随机题图和题目说明
- 录音控制组件
  - 开始、结束、重录、播放、上传五类行为
  - 包含时间显示、波形占位、录音状态标签
- Toast 提示组件
  - 用于上传成功、上传失败、分享成功、重录确认反馈
- 结果展示组件
  - CEFR 图、等级标签、对应背景图、随机文案、链接

## 5. 交互说明
- 筛选 / 搜索
  - 本期前台页面无复杂筛选和搜索
- 弹窗
  - 用户点击重新录音时弹出确认提示：之前的录音内容将被删除
  - 用户录音权限拒绝时弹出权限说明提示
- 校验
  - 未授权不得进入录音流程
  - 已提交用户不得再次进入录音提交流程
  - 录音文件不存在时不得触发上传
- 提交
  - 上传录音时按钮进入 loading 状态，避免重复点击
  - 上传成功后立即切换到等待态
- 提示反馈
  - 上传成功：显示上传成功，等待测评结果
  - 上传失败：显示上传失败，请重试
  - 分享触发：显示已调起微信分享能力
  - 活动结束：显示谢谢参与，活动结束

## 6. 状态与数据需求
- 默认 / loading / 空 / 错误 / 成功
  - 首页
    - 默认：展示活动 Banner、规则、CTA
    - loading：加载用户状态与活动配置
    - 错误：状态获取失败时展示重试入口
    - 成功：按用户状态渲染 CTA
  - 录音页
    - 默认：题图已加载，等待录音
    - loading：初始化权限和题图资源
    - 错误：权限被拒绝、题图缺失、上传失败
    - 成功：录音草稿可播放并可上传
  - 等待页
    - 默认：展示固定文案和分享入口
    - loading：查询结果状态
    - 成功：仍待评分时保持等待内容
  - 结果页
    - 默认：展示等级图、背景图、随机文案
    - 错误：结果字段异常时展示兜底卡片
- 数据字段
  - unionId
  - userStatus
  - activityStatus
  - questionSetId
  - questionImageUrl
  - recordingDuration
  - audioFileId
  - audioFileUrl
  - submissionId
  - resultStatus
  - cefrLevel
  - cefrImageUrl
  - cefrBackgroundUrl
  - randomCopy
  - sharePath
- API 依赖
  - 获取用户状态接口
  - 获取活动配置接口
  - 获取随机题图接口
  - 上传录音接口
  - 查询结果接口
  - 接收结果页素材配置接口或静态映射

## 7. 响应式说明
- Desktop
  - 用于评审展示，采用双栏或多卡片布局，左侧放主叙事内容，右侧放手机预览框
  - 原型可同时展示多个页面模块，便于产品、设计、研发对照
- Tablet
  - 缩减为上下结构，保留 Banner、状态卡片和单列手机预览
- Mobile
  - 以单列纵向布局为主
  - 录音按钮和主要 CTA 采用大尺寸按钮，保证触达范围
  - 结果页主视觉优先占首屏，链接与分享置于次级区块
